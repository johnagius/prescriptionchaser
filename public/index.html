Prescription Chaser

Find the oxygen patients you need to chase for a renewed prescription — expired or living on temporary approvals, still actively swapping cylinders, and not deceased. Your uploaded files are processed entirely in your browser and never leave it. The only thing saved to the cloud is the email corrections you choose to make, so they follow you across devices.

1

Get fresh data out of Business Central

For every code below: open the relevant BC page, press F12 to open the console, switch the context dropdown (top-left of the console, usually says top) to Multigas Production, then paste the copied code and press Enter. Each script auto-downloads a CSV when it finishes.

Customer Data

  1. In BC press Alt+Q, type customer, press Enter
  2. Open the console → select Multigas Production
  3. Paste & run → customer-contact-details CSV downloads

Expired / Temporary Approvals

  1. In BC press Alt+Q, type Prescription list, press Enter
  2. Open the console → select Multigas Production
  3. Paste & run → prescription-temp-watchlist CSV downloads

Cylinder Exchanges (last 30 days)

  1. Open Item Ledger Entries, apply the saved filter John Oxygen Exchanges
  2. Open the pivot table (switch on Analysis mode if it isn't already)
  3. Open the console → select Multigas Production
  4. Paste & run → source-cylinders-last-30-days CSV downloads

People to chase

Carers are grouped when patients share an email or a phone number, so one carer gets one combined email. Use ✎ on a carer to replace its email everywhere or add extra recipients (remembered in the cloud). Anyone emailed in the last 24h is greyed-out and skipped.
